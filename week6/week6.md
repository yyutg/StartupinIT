## 1. Уточнённое ценностное предложение

**Операционализированная формулировка:**

> «Для **активных криптотрейдеров и DeFi-пользователей 22–35 лет**, ежедневно сталкивающихся с информационной перегрузкой и тратящих 5–10 часов в неделю на анализ проектов, в ситуации **когда нужно быстро оценить новый актив и принять инвестиционное решение, но данные разбросаны по 5–10 разным сервисам**, CoinTracker обеспечивает **единое окно для глубокого анализа крипторынка: графики в реальном времени, ончейн-данные, AI-скоринг проектов и кастомные алерты**, сокращая время на анализ проекта с 15–20 минут до 2–3 минут и снижая риск попадания в скам-проекты на 70%, по сравнению с использованием DexScreener, CoinGecko, DefiLlama и других сервисов по отдельности, при условии наличия стабильного интернет-соединения.»

**Основания доверия (почему пользователь поверит):**

1. **Единое окно.** Вся аналитика в одном интерфейсе: графики, ончейн-данные (TVL, активные адреса), фундаментальные метрики, новости, скоринг. Не нужно открывать 5–10 вкладок.
2. **Прозрачный AI-скоринг.** Пользователь видит, из каких 20+ метрик складывается оценка проекта (ликвидность, TVL, ончейн-активность, команда, соцсети). Алгоритм понятен и объясним.
3. **Данные в реальном времени.** Интеграция с DexScreener, биржами и блокчейнами обеспечивает актуальность данных с задержкой < 1 секунды.
4. **Freemium-модель.** Базовый функционал (базовая информация о монетах) бесплатен — можно протестировать сервис перед переходом на платную подписку.

## 2. Основной пользовательский сценарий

**«Трейдер анализирует новый проект и принимает решение»**

| Шаг | Кто | Действие | Что происходит в приложении |
|-----|-----|----------|-----------------------------|
| 1 | Трейдер | Открывает CoinTracker → Поиск монеты | Вводит название токена или контракт, приложение показывает дашборд по активу |
| 2 | Приложение | — | Отображает единый дашборд: график цены, ончейн-метрики (TVL, активные адреса), фундаментальные данные, новости, скоринг |
| 3 | Трейдер | Изучает график (таймфреймы, индикаторы) | Интерактивный график с возможностью переключения таймфреймов, наложения индикаторов (RSI, MACD, объёмы) |
| 4 | Трейдер | Изучает ончейн-данные | Видит TVL (для DeFi-проектов), количество активных адресов, объём транзакций, распределение токенов |
| 5 | Трейдер | Изучает фундаментальный анализ | Видит информацию о команде, токеномике, аудите, социальных сетях, партнёрах |
| 6 | Приложение | — | Показывает AI-скоринг проекта (1–100 баллов) с детализацией по категориям: ликвидность, ончейн-активность, команда, соцсети, риски |
| 7 | Трейдер | Принимает решение | На основе всех данных решает войти в позицию или пропустить проект |
| 8 | Трейдер | Настраивает алерт (опционально) | Устанавливает уведомление на ценовой уровень, изменение TVL или появление новостей по проекту |

## 3. Альтернативный сценарий

**«Пользователь отслеживает свой портфель и получает алерт»**

| Шаг | Кто | Действие | Что происходит |
|-----|-----|----------|----------------|
| 1 | Трейдер | Подключает API биржи к CoinTracker | Вводит API-ключи от Bybit/OKX/HTX, приложение импортирует его портфель |
| 2 | Приложение | — | Отображает дашборд портфеля: P&L, распределение активов, риск-метрики |
| 3 | Приложение | — | При срабатывании алерта (цена упала на 5%, TVL резко вырос) отправляет push-уведомление |
| 4 | Трейдер | Получает уведомление | Открывает CoinTracker, видит детали события и рекомендацию (например, «TVL проекта X вырос на 20% за час — возможен рост») |
| 5 | Трейдер | Принимает решение | На основе анализа решает увеличить/уменьшить позицию |

## 6. Нефункциональные требования

| # | Категория | Требование | Обоснование |
|---|-----------|-----------|-------------|
| NFR-01 | Производительность | Загрузка дашборда по монете — < 2 секунд (при стабильном интернете) | Трейдеры не готовы ждать — скорость принятия решения критична |
| NFR-02 | Производительность | Задержка обновления цен и графиков — < 1 секунды для топ-100 монет, < 5 секунд для остальных | Для дейтрейдинга критична актуальность данных |
| NFR-03 | Надёжность | Доступность API (биржи, блокчейны) не должна влиять на работу платформы — кэширование данных на 5–10 минут | При падении одного источника пользователь получает кэшированные данные |
| NFR-04 | Безопасность | API-ключи бирж хранятся в зашифрованном виде (AES-256); доступ только по HTTPS; двухфакторная аутентификация для аккаунта | Защита средств пользователей — критический фактор доверия |
| NFR-05 | Безопасность | Отсутствие сбора лишних персональных данных; соответствие 152-ФЗ | Снижает барьер доверия; юридическая чистота |
| NFR-06 | Совместимость | Поддержка iOS 15+ и Android 10+ (PWA); корректный UI на экранах от 4.7 дюймов; десктопная версия с адаптивным дизайном | Охватывает > 90% актуальных устройств целевой аудитории |
| NFR-07 | Масштабируемость | Поддержка до 10 000 одновременных пользователей на серверной инфраструктуре (MVP) | Обеспечивает рост на первых этапах без перестройки архитектуры |
| NFR-08 | Доступность интерфейса | Все интерактивные элементы — минимум 44 × 44 pt; поддержка системного размера шрифта; тёмная тема для ночной торговли | Трейдеры часто работают ночью — тёмная тема обязательна |
| NFR-09 | Доступность | Время реакции системы на действия пользователя — < 500 мс (для UI), < 2 секунд (для загрузки данных) | Мгновенная обратная связь важна для активных трейдеров |
| NFR-10 | Автономность | Кэширование последних данных на устройстве; при потере интернета пользователь видит кэшированные данные с пометкой «данные могут быть устаревшими» | Трейдер может находиться в месте с нестабильным интернетом |
| NFR-11 | Архитектура | Backend: Node.js / Python + PostgreSQL + Redis; Frontend: React PWA; WebSocket для реального времени; интеграции: DexScreener API, CoinGecko API, DefiLlama API, блокчейн-ноды | Обеспечивает масштабируемость, скорость и независимость от внешних API |
| NFR-12 | Мониторинг | Логирование ошибок; метрики: количество активных пользователей, среднее время на дашборде, частота использования алертов, конверсия в платную подписку | Позволяет отслеживать здоровье сервиса и ключевые метрики роста |
