## Шаг 1. Цель MVP и метрика успеха

**Формулировка цели по шаблону:**

> «На MVP мы проверяем, что **активный криптотрейдер 22–35 лет, ежедневно анализирующий рынок**, готов **использовать единую платформу для анализа крипторынка**, потому что получает **все данные (графики, ончейн-метрики, фундаментальный анализ, AI-скоринг) в одном окне, сокращая время на анализ с 15–20 минут до 2–3 минут на проект**.
> Успех: **≥70% пользователей завершают анализ хотя бы одного проекта через дашборд**, **≥30% участников тестового запуска говорят "я бы продолжил использовать платформу"** и **≥20% готовы рассмотреть платную подписку (500–1500₽/мес)**.»

| Параметр | Значение |
|---|---|
| **Сегмент** | Активные криптотрейдеры и DeFi-пользователи 22–35 лет, с опытом от 6 месяцев, тратящие 5–10 часов в неделю на анализ |
| **Ценность (1 фраза)** | Единое окно для анализа крипторынка — графики, ончейн-данные, скоринг, новости в одном месте |
| **Целевое действие на MVP** | Завершение анализа хотя бы одного проекта через дашборд (от поиска до просмотра скоринга) |
| **Метрика 1** | Completion rate: ≥70% пользователей, начавших поиск проекта, доходят до просмотра скоринга |
| **Метрика 2** | Retention intent: ≥30% участников отвечают «да» на вопрос «хотели бы продолжить использовать платформу?» |
| **Метрика 3** | Конверсионный интерес: ≥20% участников готовы рассмотреть платную подписку (500–1500₽/мес) |

## Шаг 2. Основной user flow

**Сценарий:** Трейдер (28 лет) видит новость о новом DeFi-проекте, хочет быстро оценить его потенциал.

| № | Шаг | Действие | Результат |
|---|-----|----------|-----------|
| 1 | **Триггер** | Пользователь открывает веб-приложение CoinTracker по ссылке из Telegram-чата / рекламы | Открывается главный экран с поиском |
| 2 | **Поиск** | Вводит название токена или контракт в строку поиска | Отображаются результаты поиска (автодополнение) |
| 3 | **Выбор проекта** | Нажимает на нужный проект | Открывается дашборд проекта |
| 4 | **Просмотр графика** | Изучает график цены (переключает таймфреймы, накладывает индикаторы) | График обновляется в реальном времени |
| 5 | **Просмотр ончейн-данных** | Прокручивает до блока ончейн-метрик | Видит TVL, активные адреса, объём транзакций |
| 6 | **Просмотр фундаментальных данных** | Прокручивает до блока фундаментального анализа | Видит информацию о команде, токеномике, аудите, соцсетях |
| 7 | **Просмотр скоринга** | Смотрит AI-скоринг проекта (1–100 баллов) и детализацию по категориям | Понимает уровень риска и потенциала проекта |
| 8 | **Принятие решения** | На основе всех данных решает войти в позицию или пропустить проект | — |
| 9 | **Настройка алерта (опционально)** | Нажимает «Создать алерт» и выбирает условия | Алерт сохраняется, пользователь получит уведомление при срабатывании |

## Шаг 3. Альтернативный сценарий

**Тип:** Проект не найден в базе данных

| № | Шаг | Описание |
|---|-----|----------|
| 1 | **Триггер** | Пользователь вводит название токена или контракт, которого нет в базе данных CoinTracker |
| 2 | **Уведомление** | Приложение показывает сообщение: «Проект не найден в нашей базе. Возможно, он очень новый или это скам» |
| 3 | **Опции** | Пользователю предлагается: «Запросить добавление проекта» (уходит в очередь модерации) или «Поискать похожие проекты» |
| 4 | **Запрос на добавление** | Пользователь нажимает «Запросить добавление», вводит дополнительную информацию (сайт, контракт) | Заявка попадает в очередь модерации администратора |
| 5 | **Уведомление о готовности** | После добавления проекта в базу (1–2 дня) пользователь получает уведомление: «Проект X добавлен в базу. Можете посмотреть анализ» | Пользователь возвращается на дашборд проекта |

**Тип:** Пользователь хочет сравнить несколько проектов

| № | Шаг | Описание |
|---|-----|----------|
| 1 | **Триггер** | Пользователь находится на дашборде проекта и нажимает «Сравнить с другим проектом» |
| 2 | **Выбор проектов** | Пользователь выбирает до 5 проектов для сравнения через поиск |
| 3 | **Отображение сравнения** | Приложение отображает таблицу сравнения: цена, TVL, активные адреса, скоринг, токеномика |
| 4 | **Детализация** | Пользователь может нажать на любой показатель, чтобы увидеть график или детальное объяснение |
| 5 | **Экспорт** | Пользователь может экспортировать сравнение в PDF/Excel |

| **6. AI-скоринг (виджет)** | Шаг 7 (скоринг) | Шкала 0–100 баллов, детализация по категориям (ликвидность, ончейн-активность, команда, соцсети, риски), пояснение метрик | Загрузка / Оценка загружена | — |
| **7. Сравнение проектов** | Альтернативный сценарий | Таблица сравнения: до 5 проектов, ключевые метрики (цена, TVL, активные адреса, скоринг), кнопка экспорта | Пусто / Загрузка / Данные загружены | → Экспорт |
| **8. Настройка алерта** | Шаг 9 (алерт) | Выбор метрики (цена, TVL, новости), пороговое значение, способ уведомления (push/Telegram) | Настройка / Сохранено | → Дашборд проекта |
| **9. Экран ошибки** | Альтернативный сценарий | Сообщение об ошибке (проект не найден, нет данных, технический сбой), кнопка «Попробовать снова» или «Запросить добавление» | — | → Главный экран |

## Шаг 7. Definition of Done для MVP

MVP считается готовым, если выполнены все следующие критерии:

1. **Реализованы все Must-истории** (US-01 — US-05, US-07) и прошли ручное тестирование на реальных проектах
2. **Основной user flow проходит без блокирующих ошибок**: поиск проекта → дашборд → просмотр скоринга завершается корректно в ≥90% тестовых запусков
3. **Данные обновляются в рамках таймингов**: загрузка дашборда ≤ 2 секунд, обновление цен ≤ 1 секунда для топ-100 монет
4. **Обработаны ключевые ошибки**: проект не найден, нет данных по API, таймаут загрузки, технический сбой
5. **Логика AI-скоринга верифицирована**: скоринг для 50 тестовых проектов проверен экспертами, точность > 70%
6. **Сохранность данных**: пользовательские алерты и избранное сохраняются в БД и не теряются при перезапуске сервера
7. **Настроен сбор минимум 3 метрик**: (1) процент завершённых анализов (дошли до скоринга), (2) средняя оценка полезности платформы (1–5), (3) готовность повторить / рассмотреть платную подписку
8. **Себестоимость сессии подтверждена**: анализ одного проекта (запросы к 3–5 API) обходится ≤ 2₽ за пользователя
9. **Подготовлен сценарий демо**: пошаговый скрипт демо-анализа 3 проектов (DeFi, мемкоин, инфраструктура) на 15–20 минут
10. **Задокументировано**: инструкция запуска веб-приложения для тестовой группы — 1 страница (как открыть, как искать проекты, как пользоваться скорингом)
11. **Зафиксированы ограничения MVP**: список всего, что **не входит** в MVP (мобильное приложение, торговля через интерфейс, социальная сеть трейдеров, API для ботов, децентрализованная версия)
12. **Проведён минимум 1 тестовый плейтест** с реальными пользователями (≥5 человек из целевого сегмента) и зафиксированы результаты

## Шаг 8. План демо / пилота

### Кому показываем

5–10 контактов из целевого сегмента:

| # | Профиль | Формат |
|---|---------|--------|
| 1–3 | Активные криптотрейдеры из TG-чатов (@crypto_signals, @defi_russia), с опытом от 6 месяцев | Онлайн-демо |
| 4–6 | DeFi-пользователи, участвующие в стейкинге и фарминге | Онлайн-демо |
| 7–8 | Инвесторы из криптосообществ (Reddit, Twitter), согласившиеся протестировать | Онлайн-демо |
| 9–10 | Аналитики из крипто-фондов или крупные трейдеры (для экспертной оценки) | Экспертное интервью |

### Как показываем

- **Онлайн-демо**: созвон в Telegram/Zoom, демонстрация веб-приложения, пользователь проходит flow от поиска до просмотра скоринга на 2–3 проектах
- **Для экспертов**: демо расширенных функций (сравнение, алерты) + обсуждение точности скоринга

### 5 вопросов для пользователей

1. «Что тебе понравилось / не понравилось в процессе?» *(открытый, без наводки)*
2. «Насколько удобно иметь все данные (график, ончейн, фундаментал) в одном окне? Это лучше, чем твой текущий процесс?» *(ключевая гипотеза H1 — единое окно)*
3. «Доверяешь ли ты AI-скорингу? Что нужно изменить, чтобы доверие выросло?» *(гипотеза H3 — точность скоринга)*
4. «Хотели бы вы продолжить использовать платформу в будущем? Почему да / нет?» *(гипотеза H2 — retention)*
5. «Если бы Pro-подписка стоила 500–1500₽/мес и давала больше алертов и расширенный скоринг — рассмотрели бы? Почему?» *(гипотеза H2 — конверсия в платные)*

### Что считаем успехом

| Сигнал | Порог |
|--------|-------|
| Completion rate (дошли до скоринга) | ≥70% |
| Удобство единого окна (оценка 1–10) | ≥8 |
| Готовность продолжить использование | ≥30% |
| Готовность рассмотреть платную подписку | ≥20% |
| Технические сбои (задержки, ошибки API) как проблема | <20% респондентов |
