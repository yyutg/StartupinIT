# Аудит MVP проекта CoinTracker

## Шаг 1. Контроль соответствия MVP исходным гипотезам

### Мини-чек

- **Сегмент:** активные криптотрейдеры и DeFi-пользователи 22–35 лет, с опытом от 6 месяцев, которые тратят 5–10 часов в неделю на анализ проектов и используют 3–10 разных сервисов.
- **Проблема:** данные разрознены (DexScreener, CoinGecko, DefiLlama, блокчейн-обозреватели), анализ проекта занимает 15–20 минут, высок риск попадания в скам-проекты.
- **Ценность (value proposition):** единое окно для глубокого анализа крипторынка — графики, ончейн-данные, фундаментальный анализ, AI-скоринг проектов и кастомные алерты.
- **Целевое действие на MVP:** пользователь находит проект через поиск, открывает дашборд и доходит до просмотра AI-скоринга.
- **Что считаем «успехом»:** конверсия из поиска в просмотр дашборда (CTR на выбранный проект) и доля пользователей, дошедших до блока скоринга.

### Каким образом MVP решает заявленную проблему?

Текущий MVP хорошо закрывает информационную часть проблемы: пользователь может найти проект, увидеть график цены, ончейн-метрики и базовый AI-скоринг. MVP демонстрирует ценность «единого окна» — не нужно открывать 5–10 вкладок. При этом MVP пока не реализует кастомные алерты и портфельный трекер (планируются в следующих версиях), а также API-интеграция с некоторыми биржами может работать с задержками.

### Соответствует ли продукт первоначальной концепции и гипотезам ЦА?

Да, по концепции проект соответствует гипотезе «единая платформа для анализа крипторынка», и это явно отражено в структуре дашборда и наборе метрик. Гипотеза о ценности «сэкономить время и снизить риск скамов» также поддерживается через AI-скоринг и агрегацию данных. Но проверка ключевой гипотезы поведения ЦА («готовы платить за расширенную аналитику») пока ограничена, так как нет реализации платной подписки в MVP.

## Шаг 2. Аудит функциональности и юзабилити

| Функция | Категория (Core/Support/Later) | Какой шаг сценария обеспечивает | Вклад в ценность | Проблемы/Замечания |
|---|---|---|---|---|
| Поиск проектов с автодополнением | Core | Вход в сценарий и поиск проекта | Быстрый доступ к нужному активу | Хорошо работает, можно добавить поиск по контракту |
| Дашборд проекта (единое окно) | Core | Основной анализ проекта | Главная ценность продукта — вся аналитика в одном месте | Нужно ускорить загрузку (сейчас 2–3 секунды) |
| График цены с таймфреймами и индикаторами | Core | Технический анализ | Оценка ценовой динамики и трендов | Хорошо работает, но не хватает большего количества индикаторов |
| Ончейн-метрики (TVL, активные адреса, объём транзакций) | Core | Глубокий анализ активности | Понимание реального использования проекта | Данные обновляются с задержкой 5–10 минут |
| Фундаментальные данные (команда, токеномика, аудит, соцсети) | Core | Оценка качества проекта | Снижение риска скамов | Не у всех проектов данные заполнены полностью |
| AI-скоринг (1–100 баллов с детализацией) | Core | Быстрая оценка рисков и потенциала | Ключевое преимущество перед конкурентами | Нужно добавить пояснения по каждой метрике |
| Кнопка «Сравнить с другим проектом» | Support | Сравнительный анализ | Помогает выбрать лучший проект для инвестиций | Пока работает только для 2 проектов (MVP) |
| Создание алерта (цена, TVL) | Core (частично) | Отслеживание событий без постоянного мониторинга | Экономия времени, своевременное реагирование | В MVP реализованы только базовые алерты (без новостей) |
| Портфельный трекер (интеграция с API бирж) | Later | Отслеживание P&L | Удобство управления портфелем | Не реализован в MVP |
| Избранное (сохранение проектов) | Support | Быстрый доступ к отслеживаемым проектам | Удобство повторного анализа | Реализован базово, без синхронизации |
| Тёмная тема | Support | Комфорт при ночной торговле | Улучшение UX для активных трейдеров | Есть, но не везде корректно отображается |
| Footer с соц-кнопками | Later | Доп. коммуникация | Низкий вклад в целевой сценарий | Кнопки есть, но ссылки пока не ведут на рабочие каналы |

### Какие фичи можно убрать без потери ценности

- Footer с соц-кнопками, если они пока не ведут на рабочие каналы поддержки и сообщества.
- Избыточные декоративные элементы на главной странице (анимации, увеличивающие время загрузки).
- Часть расширенных фильтров в поиске (для MVP достаточно базового поиска).

### Какие фичи стоит упростить

- Дашборд проекта: сначала показывать ключевые метрики (график, скоринг, TVL), остальное — под спойлером или по клику.
- Ончейн-метрики: для MVP достаточно 3–4 ключевых показателей (TVL, активные адреса, объём транзакций).
- Форму создания алерта: упростить до выбора метрики и порогового значения (без сложных условий).

### Какие фичи нужно добавить, если без них ключевой сценарий «ломается»

- Рабочее действие кнопки «Создать алерт» (с сохранением в БД и базовой проверкой срабатывания).
- Пояснения к AI-скорингу (при наведении на категорию — всплывающая подсказка с пояснением метрики).
- Базовая метрика событий (хотя бы клики по поиску, выбору проекта, просмотру скоринга), чтобы измерять успех MVP.

## Шаг 3. Корректировка функциональности

### 1–3 функции убрать/перенести в Later

1. Перенести **портфельный трекер** (интеграция с API бирж) в Later — это сложная интеграция, не критичная для проверки основной гипотезы о ценности единого окна.
2. Убрать декоративные элементы футера (иконки-ссылки), пока нет рабочего канала коммуникации с пользователями.
3. Сократить количество расширенных фильтров в поиске — для MVP достаточно поиска по названию и символу.

### 1–3 функции добавить в Core MVP (без которых ценность не достигается)

1. **Пояснения к AI-скорингу** — при наведении на категорию (ликвидность, ончейн, команда, соцсети, риски) должна появляться всплывающая подсказка с пояснением, из чего складывается оценка. Это повышает доверие к скорингу.
2. **Базовый алерт по цене** — пользователь должен иметь возможность создать алерт на превышение/падение цены и получить уведомление (push или Telegram). Без этого ключевая ценность «не пропустить возможность» не достигается.
3. **Индикатор загрузки данных** — при запросе к API (график, ончейн-данные) должен отображаться скелетон или спиннер, чтобы пользователь понимал, что данные загружаются, а не сломались.

### 1–3 функции упростить

1. **Дашборд проекта** — вынести блоки с метриками в аккордеоны или табы, чтобы сначала показывать только самое важное (график, скоринг, TVL), а остальное — по желанию пользователя.
2. **Форма создания алерта** — оставить только базовые поля: выбор метрики (цена / TVL), условие (выше / ниже), пороговое значение. Без сложных условий и комбинаций.
3. **Ончейн-метрики** — для MVP оставить только TVL, активные адреса и объём транзакций. Распределение токенов (топ-10 холдеров) можно добавить позже.

## Шаг 4. План тестирования MVP

### Что тестируем

| Гипотеза | Метод проверки | Критерий успеха |
|---|---|---|
| **H1:** Пользователи готовы использовать единую платформу вместо 3–10 разных сервисов | Опрос после тестирования: «Сколько сервисов вы использовали раньше? Удобнее ли с CoinTracker?» | ≥70% говорят, что CoinTracker удобнее текущего набора сервисов |
| **H2:** AI-скоринг помогает быстрее оценивать проекты и снижает риск скамов | Сравнение времени анализа проекта с CoinTracker vs без него. Опрос о доверии к скорингу | Сокращение времени анализа на 50%; ≥60% доверяют скорингу |
| **H3:** Пользователи готовы платить 500–1500₽/мес за расширенные функции | Опрос о готовности платить после тестирования базовой версии | ≥20% готовы рассмотреть платную подписку |

### Кого привлекаем

| # | Профиль | Источник | Формат |
|---|---------|----------|--------|
| 1–3 | Активные криптотрейдеры (опыт 1–3 года) | TG-каналы @crypto_signals, @defi_russia | Онлайн-демо + опрос |
| 4–6 | DeFi-пользователи (стейкинг, фарминг) | TG-каналы, Discord-серверы | Онлайн-демо + опрос |
| 7–8 | Инвесторы из криптосообществ | Twitter, Reddit r/CryptoCurrency | Онлайн-демо + опрос |
| 9–10 | Аналитики / опытные трейдеры (для экспертной оценки) | Личные контакты, профессиональные сообщества | Экспертное интервью |

### Сценарий тестирования

1. **Открыть CoinTracker** → найти проект по названию (например, Uniswap).
2. **Изучить дашборд** → график, ончейн-метрики, фундаментальные данные, AI-скоринг.
3. **Сравнить с проектом-конкурентом** (PancakeSwap) через кнопку «Сравнить».
4. **Создать алерт** по цене (например, «уведомить, если цена упадет на 5%»).
5. **Ответить на вопросы** (5–7 минут).

### Метрики успеха

| Метрика | Целевое значение |
|---------|------------------|
| Completion rate (дошли до просмотра скоринга) | ≥70% |
| Среднее время анализа одного проекта | ≤3 минут |
| Доля пользователей, которые сказали «удобнее, чем раньше» | ≥70% |
| Готовность продолжить использование | ≥30% |
| Готовность рассмотреть платную подписку | ≥20% |
| Технические сбои (задержки, ошибки API) как проблема | <20% респондентов |

## Шаг 6. Выводы и следующие шаги

### Что сделано хорошо

1. **Единое окно** — MVP успешно агрегирует данные из 5+ источников, пользователь получает всю аналитику в одном месте.
2. **AI-скоринг** — реализована базовая версия скоринга, которая даёт пользователю быструю оценку проекта.
3. **График и ончейн-данные** — ключевые метрики для трейдера и DeFi-пользователя присутствуют и обновляются в реальном времени.

### Что нужно доработать до тестирования

1. **Пояснения к скорингу** — добавить всплывающие подсказки для каждой категории (ликвидность, ончейн, команда, соцсети, риски).
2. **Базовые алерты** — реализовать создание алерта по цене с отправкой push-уведомления.
3. **Оптимизация загрузки** — ускорить загрузку дашборда до < 2 секунд (кэширование, параллельные запросы).

### Риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| API бирж/блокчейнов недоступны | Средняя | Высокое | Кэширование данных на 5–10 минут, fallback-источники |
| AI-скоринг неточен | Средняя | Высокое | Прозрачность метрик, возможность жалобы на неточность, итеративное улучшение |
| Пользователи не готовы платить | Средняя | Среднее | Freemium-модель, бесплатный базовый функционал |
| Низкий retention | Высокая | Среднее | Алерты и дайджесты для возврата пользователей |

### Следующие шаги

1. **Доработка MVP** (2–3 дня): добавить пояснения к скорингу, реализовать базовые алерты, оптимизировать загрузку.
2. **Тестирование на 10 пользователях** (3–5 дней): собрать обратную связь, замерить метрики.
3. **Анализ результатов** (1–2 дня): подтвердить/опровергнуть гипотезы, сформировать план следующей итерации.
4. **Презентация результатов** (1 день): подготовить отчёт и демо для инвесторов/акселератора.
